Maven Central Repositoryへの公開に必要な、GPG署名用の鍵（認証の鍵）の作成および設定手順を説明します。

GnuPG (GPG) を使用して鍵を作成し、その情報をプロジェクトの設定に反映させます。

### 1. GnuPGのインストール
まだインストールされていない場合は、GnuPGをインストールしてください。
- **macOS (Homebrew):** `brew install gnupg`
- **Windows:** [Gpg4win](https://gpg4win.org/) 等をインストール

### 2. GPG鍵の作成
ターミナルで以下のコマンドを実行します。

```bash
gpg --full-generate-key
```

以下の項目を聞かれるので、適宜選択・入力してください。
- **種類:** `RSA and RSA (default)` を選択（または `1`）
- **鍵のサイズ:** `4096` を推奨
- **有効期限:** `0` (無期限) または任意の期間
- **名前:** あなたの名前
- **電子メール:** あなたのメールアドレス
- **パスフレーズ:** **重要！** 鍵を保護するためのパスワードです。忘れないように控えておいてください（`signing.password` に使用します）。

### 3. 公開鍵をサーバーに配布
Maven Centralの検証システムが署名を検証できるように、公開鍵をキーサーバーに送信します。

まず、作成した鍵のIDを確認します。
```bash
gpg --list-keys
```
`pub   rsa4096 2026-01-07 [SC]` の下の行にある、長い英数字の末尾8文字（または全体）がキーIDです。

次に、そのIDを使ってサーバーに送信します（例：keyserver.ubuntu.com）。
```bash
gpg --keyserver keyserver.ubuntu.com --send-keys [あなたのキーID]
```

### 4. 秘密鍵ファイルの書き出し
Gradleの `signing` プラグインで利用するために、秘密鍵（セクレタリーリング）をファイルとして書き出します。

```bash
gpg --export-secret-keys -o secring.gpg
```
※ この `secring.gpg` は、`signing.secretKeyRingFile` として指定するファイルになります。プロジェクトのルートディレクトリ（`.gitignore` で除外されていることを確認してください）や、安全な場所に保存してください。

### 5. `gradle.properties` への設定
作成した情報を、プロジェクトの `gradle.properties` に反映させます。

```properties
# Sonatype (Maven Central) の認証情報
ossrhUsername=あなたのSonatypeユーザー名
ossrhPassword=あなたのSonatypeパスワード

# GPG署名の設定
signing.keyId=あなたのキーID（末尾8文字）
signing.password=鍵作成時に設定したパスフレーズ
signing.secretKeyRingFile=secring.gpg への絶対パス（またはプロジェクトルートからの相対パス）
```

### 注意事項
- **セキュリティ:** `gradle.properties` や `secring.gpg` には機密情報が含まれます。これらを絶対に変更履歴（Git等）にコミットしないように注意してください。
- **Git除外:** `.gitignore` に `secring.gpg` や、機密情報を含む場合の `gradle.properties` が含まれているか確認してください。
- **最近のGPG:** 最新のGPGでは `secring.gpg` の形式がデフォルトで書き出せない場合があります。その場合は `gpg --export-secret-keys > secring.gpg` などの方法を検討してください。
